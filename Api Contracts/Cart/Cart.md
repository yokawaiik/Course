# Синхронизация корзины с сервером.

## Логика синхронизации

1. Если пользователь не авторизован (при регистрации, при входе в аккаунт)
При авторизации делаем merge клиентского состояния корзины с серверным (складываем количество одинаковых товаров и добавляем новые в корзину) (зависит от бизнес-требований)

2. Если пользователь авторизован, но был без сети и сеть восстановлена
Сервер сверяет last_known_revision. Если на сервере уже ревизия 145 (пока клиент был оффлайн, жена пользователя с другого устройства изменила корзину), сервер производит авто-мерж по бизнес-правилам и возвращает клиенту актуальное состояние и новую ревизию.

3. Увеличиваем last_known_revision на сервере когда делаем изменения в состоянии корзины при условии онлайна или восстановления
