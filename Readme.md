# Architectural Exercises

Small architectural exercises implementing FSD + Clean Architecture patterns in Dart/Flutter.

## Structure

- **Api Contracts/** — API contract designs and data layer implementations
  - `Cart/` — Cart sync logic (merge strategies, revision handling)
  - `State WebSocket/` — WebSocket client abstraction with connection state management

## Topics

- Feature-Sliced Design (FSD) module organization
- Clean Architecture layer separation (domain / data / presentation)
- API contract definition and DTO mapping
